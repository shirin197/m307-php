Wie funktionierts:
(Buch Seite 2; Abb. 1-1)
Serverseitig:
- heisst, dass der PHP-Code nicht aif dem Client(Browser), ausgeführt wird sondern auf den Server.
Kommunikation im Internet bei statischen HTML-seiten ohne PHP:
- Browser als Webclient -> Anfrage (REQUEST) an den Server (Webserver-Software) nach der Datei. 
- Server liefert als Antwort (RESPONSE) diese Datei and den Browser, der sie dann darstellt.

Mit PHP:
 1. Sufer gib eine Adresse ein. Der Browser leitet den REQUEST and den Webserver weiter.
 2. Bei der angeforderten Seite handeltes sich disemal um ein PHP-SCRIPT. Der Server erkennt die Endung .php. Desswegen wird die Seite
    auf ein Programm - bei PHP dem PHP-Praser übergeben, nicht an Broswer (Client).
 3. Die PHP-Befehle werden vom Praser intepretiert und es wird eine neue HTML-Seite erzeugt.
 4. Die HTML-Seite wird and en Browser zurückgesendet.  Die angezeigte Seite heisst zwar och xy.php, aber enthält keinen PHP-Code, sondern nur 
    HTML-Code.
- PHP steht für -> PHP Hypertext Preprocessor
